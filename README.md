# AI Systems Weekly

`ai-systems-weekly` 是一个面向公开发布的 AI Systems / Agent 内容仓库。

它不是单一脚本项目，而是一个持续沉淀的内容资产库：用来记录日常观察、阶段性周刊、选题池、模板和写作素材，适合做长期更新的技术内容仓库、知识管理仓库或公开写作工作台。

## 仓库定位

- 公开沉淀 AI Systems、Agent、工作流和行业观察相关内容
- 同时保留写作过程材料与面向发布的结构化输出
- 适合持续更新，而不是一次性交付

## 仓库包含什么

- 日志与日更内容
- 周刊与阶段总结
- 选题池与写作线索
- 面向后续写作复用的模板
- 面向发布使用的脚本和目录结构

## 目录结构

```text
ai-systems-weekly/
├─ docs/            # 面向发布的结构化内容目录
├─ 日志/             # 写作日志与过程记录
├─ 期刊/             # 阶段性内容整合输出
├─ 选题池/            # 候选选题、草稿与写作方向
├─ 模板/             # 日志、期刊、论文卡片、评测模板
├─ scripts/         # 内容维护脚本
└─ publish.bat      # 发布辅助脚本
```

## 适合谁

- 想系统记录 AI 研究与产品观察的人
- 想把“日志、选题、周刊、发布”放到同一仓库管理的人
- 想维护公开技术写作仓库的人

## 使用方式

1. 把它当作内容母仓库，持续追加日志与选题
2. 用 `模板/` 和 `docs/` 里的结构保持输出一致性
3. 逐步把阶段性结论沉淀成周刊、专题或长期专栏

## 说明

- 这个仓库更偏内容管理，不强调传统软件项目那种“启动即运行”
- 如果你只想看最终输出，优先看 `docs/`
- 如果你想看选题与写作演进过程，可以看 `日志/`、`期刊/`、`选题池/`
